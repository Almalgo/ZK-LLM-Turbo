# T3.1 OpenFHE MatMul Implementation Plan

Date: 2026-04-12

## Scope

Implement `common/he_backend.py::matmul()` for the OpenFHE backend so server inference can run without TenSEAL-specific `.mm()`.

## Constraints

- OpenFHE now imports in this environment, but only smoke-level validation has been completed.
- Implementation must be validated in a known-good OpenFHE runtime before enabling by default.
- Keep TenSEAL path unchanged and as the fallback.

## Current Implementation

1. Encode each output-column vector from the plaintext matrix as CKKS plaintext.
2. Compute per-output ciphertexts using `EvalInnerProduct`.
3. Pack per-output ciphertexts into one ciphertext with `EvalMerge`.
4. Reuse generated EvalSum/rotation keys via context metadata.

## Next Improvements

1. Add larger-dimension benchmark coverage and compare against TenSEAL baseline.
2. Validate memory overhead from OpenFHE key generation at realistic output widths.
3. Evaluate whether chunked merge strategy is required for very large output dimensions.
4. Keep TenSEAL fallback as default until parity/performance gates are met.

## Verification Plan

1. Unit tests with a fake OpenFHE module for call-shape and metadata handling.
2. Backend smoke benchmark via `benchmarks/bench_he_matmul_backend.py` (small dimensions).
3. Real OpenFHE parity test:
   - small matrix (8x4)
   - hidden-size matrix (2048x256)
   - full hidden matrix (2048x2048)
4. Accuracy check against TenSEAL baseline decryption tolerance.
5. Runtime benchmark with `benchmarks/bench_he_matmul.py`.

## Exit Criteria

- `matmul()` no longer raises `NotImplementedError` for OpenFHE.
- `server/tests/test_he_ops.py` passes with OpenFHE selected in a working environment.
- Benchmark evidence is captured under `benchmarks/results/` and referenced from the optimization action plan.
