# T3.2 Diagonal Packing Status

Date: 2026-03-29

## What landed

- Added cyclic diagonal extraction helpers to [`server/model/weight_manager.py`](../server/model/weight_manager.py)
- Added caching for per-layer diagonal weights via `get_layer_diagonal_weights()`
- Added unit tests for diagonal extraction and caching

## What is still blocked

The runtime packed-matmul path is still blocked on a working backend migration from `T3.1`.

Reason:

- TenSEAL's `CKKSVector.mm()` does not expose the manual rotate / multiply / accumulate control needed for THOR-style diagonal packing.
- The available `openfhe` wheel in this environment installs but does not import, so the backend that would make the manual packed matmul possible is not ready yet.

## Conclusion

`T3.2` has landed its preprocessing groundwork, but not the runtime diagonal packed matmul implementation. Completing the task requires a working OpenFHE backend or another HE backend with explicit rotate/eval APIs.
