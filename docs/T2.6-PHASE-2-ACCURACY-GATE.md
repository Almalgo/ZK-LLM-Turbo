## Key Metrics

From [`benchmarks/results/phase2_accuracy_gate.json`](../benchmarks/results/phase2_accuracy_gate.json):

- Reference client `poly_silu` (degree 6) vs exact SiLU on `[-5, 5]`
  - MAE: `0.0123`
  - Max abs error: `0.0404`
- HE-safe merged FFN SiLU approximation (quadratic) vs exact SiLU on `[-5, 5]`
  - MAE: `0.1397`
  - Max abs error: `0.3347`
- Small-dimension merged FFN HE pipeline vs its plaintext quadratic target
  - Mean MAE over 3 samples: `1.94e-08`
  - Mean max abs error over 3 samples: `4.17e-08`

## Interpretation

- The `N=16384` CKKS configuration is functionally sound in the current test coverage.
- The merged FFN path is numerically correct relative to its implemented HE target.
- The limiting factor is model approximation quality, not CKKS noise:
  - the merged server path must use a quadratic SiLU approximation to stay within the current TenSEAL scale budget
  - that quadratic approximation is materially less accurate than the degree-6 client reference polynomial

## Gate Decision

Decision: **Conditional go**

- Go for continuing the engineering roadmap into Phase 3.
- Not a go for claiming model-quality parity yet.

## Remaining Risk

The original action plan asked for a 100-sentence perplexity delta and prompt-level encrypted-vs-plaintext comparisons across multiple layers. Those evaluations are not yet automated in this repo. Phase 2 is therefore cleared for continued systems work, but not fully cleared as a final quality sign-off.
