# T3.4 Full Polynomial Model Status

Date: 2026-03-29

## Current polynomial coverage

Implemented today:

- SiLU approximation
  - degree-6 client reference polynomial
  - quadratic HE-safe server polynomial for merged FFN

Not implemented yet:

- RMSNorm polynomial approximation
- Softmax polynomial / PowerFormer-style replacement
- Server-side RoPE replacement integrated into a fully homomorphic all-layer flow

## Blocking issue

The full polynomial model depends on the backend migration from `T3.1`.

Why:

- TenSEAL remains the active backend
- OpenFHE is not importable in the current environment
- without a working post-TenSEAL backend, there is no credible path to server-side full-layer polynomial execution at the scope Phase 3 expects

## Risk summary

Even after the backend blocker is removed, softmax remains the highest-risk approximation problem in the entire roadmap. The repo is not yet in a state where a meaningful perplexity target can be claimed for a full polynomial model.

## Conclusion

`T3.4` remains blocked. The only production-adjacent polynomial path in the repo today is the merged FFN SiLU approximation introduced in Phase 2.
