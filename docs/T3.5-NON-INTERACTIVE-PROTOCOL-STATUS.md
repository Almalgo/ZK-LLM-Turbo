# T3.5 Non-Interactive Protocol Status

Date: 2026-03-29

## Goal

Reduce the protocol to:

1. client sends encrypted embeddings
2. server processes the full encrypted stack
3. server returns encrypted logits

## Why it is blocked

This depends on both:

- `T3.3` GPU-accelerated CKKS
- `T3.4` full polynomial model

Neither prerequisite is ready:

- there is no working OpenFHE backend or GPU runtime in the current environment
- the repo does not yet have polynomial replacements for RMSNorm and softmax

## Current protocol state

The repo still uses the interactive split protocol from earlier phases:

- encrypted linear subgraphs on the server
- non-linear work on the client
- multiple round-trips per layer

## Conclusion

`T3.5` cannot move beyond design status until both backend acceleration and full server-side polynomial coverage exist.
