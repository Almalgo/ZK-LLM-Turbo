# T3.1 OpenFHE Environment Finding

Date: 2026-03-29

## Summary

The repo now has a backend abstraction layer in [`common/he_backend.py`](../common/he_backend.py), but the actual OpenFHE backend migration is blocked in this environment.

## What was tried

- Installed `openfhe` into the project `.venv`
- Attempted `import openfhe`

## Result

The package installs, but import fails immediately:

```text
ModuleNotFoundError: No module named 'openfhe.openfhe'
```

This indicates the published wheel available to this environment is missing its native extension module for the current interpreter/platform combination.

## Impact

- TenSEAL remains the active backend.
- The new abstraction layer allows the runtime to stop depending directly on TenSEAL primitives in the core modules.
- A real OpenFHE backend implementation still requires a working OpenFHE build or a compatible wheel.

## Next step

To finish `T3.1`, use one of:

- a compatible Python version / OpenFHE wheel combination
- a source build of OpenFHE bindings
- a containerized environment where OpenFHE is known-good
